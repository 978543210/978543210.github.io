Markdown语法及样式测试

Markdown 的设计哲学

Markdown 的目标是实现“易读易写”，
不过最需要强调的便是它的可读性。一份使用Markdown格式撰写的文件应该可以直接以纯文字发布，而且看起来不会像是由许多标签或是格式指令所构成。
Markdown 的语法有个主要的目的：用来作为一种网络内容的写作用语言。

说明

  • 在文字下方加 === 可使上一行文字变成一级标题
